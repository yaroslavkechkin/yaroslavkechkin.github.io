Why do i need a Virtual Lab

As a curious computer scientist who is always keen to test out the latest Operating System, application, or learn something new, I always used to find it a bit of a challenge to get my test environments going using only physical hardware. Sometimes you may be limited by the hardware you have at your disposal, or perhaps you don’t have any spare servers to use. In these cases, I found find setting up my own virtualized test environment will be extremely useful for my development, testing, and evaluation purposes.
There are a quite a few virtualization products out there, such as VMware Server, VMware ESXi, VMware Workstation Pro, Microsoft's Hyper-V and Sun’s VirtualBox, to name just a few. When i first started this project i had access to VMware Workstation Pro license from university and decided to stick with it, because i did not have an extra server powerful enough to run ESXi and several virtual machines at a time, and Hyper-V was not as good back then as VMware Workstation Pro.
To start, I installed VMware workstation on my laptop - Asus ROG G750JH running Microsoft Windows 8.1 Pro (Windows 10 now), loaded with Intel Core I7 2.4 GHz (3.2 GHz Boost) and 24 GB of DDR3 RAM. It is a powerfull machine capable of running multiple virtual machines with ease.
Over the years, the lab was used to host webserver & database server, evaluate applications, develop software, practice penetration testing & system security, practice reverse engineering & malware analysis, web application design, and Windows & Linux system administration.

Virtual Mchines and their functions

Windows Server 2016 - Active Directory (AD) Domain Controller, DNS Server, firewall, network monitoring. This virtual machine has connection to external switch and manages Internet connections from other machines. AD was setup to simplify user creation and managing user permissions on the network.

Penetration Testing Network Segment

PFSense - router and firewall to separate this network segment from others.
Kali Linux - for testing networks, web applications, and security of other machines.
Windows 7 - practice penetration testing and hardening of the Windows operating system.
Windows XP SP3 - practice penetration testing and hardening of the Windows operating system.
Metasploitable - practice penetration testing.
Ubuntu - practice penetration testing and hardening of the Linux operating system.

Reverse Engineering & Malware Analysis Segment

PFSense - router and firewall to separate this network segment from others.
Windows Server 2012 R2 - has IDA Pro with Hexrays, Visual Studio 2017 (Earlier versions are available if needed), Volatility Framework, Wireshark and other software neede to disect the malware executables.
Windows Server 2008 R2 - used as Command and Control center for botnet testing.
Windows 7 - test the behavior of the malware. Some monitoring software like Process Monitor is installed.
Windows XP SP3 - test the behavior of the malware. Some monitoring software like Process Monitor is installed.

Software Testing and Code Development Segment

PFSense - router and firewall to separate this network segment from others.
Windows 10 Education - used primarily for code & web development. Has Visual Studio with Android development tools, SQL, IIS, Sublime text editor, TeX Studio, and software for graphics design and editing.
Windows 10 Education - another version of Windows 10 is used to install and test updates before installing them on main system & software testing (different browsers, editors, media players, etc.). It is also used to test Python or PowerShell scripts, play with registery settings, firewall and services settings, Group Policy settings, and general hardening of the system.

Recent Changes

The project was started in September 2013 with only 3 machines on the network. It has expanded since then into full blown home laboratory suited for multitude of tasks. Recently, the server, which was holding AD on the network, was upgraded from Windows Server 2012 R2 to Windows Server 2016. Several VMs were upgraded from Windows 7 and 8.1 to Windows 10. Additionally the whole network was migrated from VMware Workstation 12 Pro to Hyper-V. With this change from Type 2 to Type 1 Hypervisor, I feel, virtual machines are running smoother and put less strain on the hardware, which makes running multiple virtual machines and performing tasks a lot easier.
